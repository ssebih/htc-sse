1. Écrivez un algorithme qui permet d’échanger les valeurs de deux variables A et B. Les valeurs initiales des

variables A et B doivent être entrées par l’utilisateur, et les contenus des variables A et B soient affichés à

l’écran après leur inversion. Implémentez-le sous Javascool

⊲ On ne peut pas ici utiliser :

a prend la valeur de b

b prend la valeur de a

... les deux variables a et b seraient alors toutes deux égales à la valeur initiale de b.

⊲ On peut utiliser une troisième variable c, ou plutôt temp pour lui donner un nom explicite. Ce qui donne

en javascool :

void main() {

int a = readInteger("Saisissez le nombre a");

int b = readInteger("Saisissez le nombre b");

println("a fait "+a+" et b fait "+b);

println("Echangeons leurs valeurs.");

int temp=a;

a=b;

b=temp;

println("a fait maintenant "+a+" et b fait maintenant "+b);

}

⊲D’autres techniques fonctionnaient, comme cet algorithme, joli, qui ne fait pas appel à une troisième

variable, mais qui n’est pas forcément plus rapide :

a prend la valeur de a-b

b prend la valeur de a+b

a prend la valeur de b-a

2. Écrivez un algorithme qui demande un nombre à l’utilisateur puis calcule et affiche le carré de ce nombre.

⊲ Dans le code ci-dessous, les commentaires sur une ligne débutent par // et ceux sur plusieurs lignes

commencent par /* et finissent par */. Le compilateur n’interprétera pas les commentaires.

void main() {

double x = readDouble("Saisissez un nombre");

/*On est obligés de changer de type, le type

int étant trop restrictif pour cette question*/

double y = pow(x, 2); // On aurait évidemment pu utiliser y=x*x

println("Le carré de " + x + " est " + y);

}

⊲ Le type double est-il un type réel ? Pas vraiment. Nous traiterons ceci plus tard. Pour s’en rendre compte,

regardez cette exécution surprenante : Le carré de -5.2 est 27.040000000000003 ... A suivre !

6. Écrivez un algorithme qui demande deux nombres à l’utilisateur et l’informe ensuite si leur produit est

négatif, positif ou nul. Attention, on ne doit toutefois pas calculer le produit.

⊲ Pour faire court, on peut rédiger des conditions complexes :

void main() {

int a = readInteger(), b = readInteger();

if (a == 0 || b == 0) println("Le produit est nul.");

else {

if ((a < 0 && b < 0) || (a > 0 && b > 0)) println("Le produit est positif .");

// si les deux sont négatifs ou les deux sont positifs

else println("Le produit est negatif .");

}

}

7. Écrivez un algorithme qui demande l’âge d’un enfant à l’utilisateur et affiche ensuite sa catégorie :

• « Poussin » de 9 à 10 ans,

• « Benjamin » de 11 à 12 ans,

• « Minime » de 13 à 14 ans,

• « Cadet » de 15 à 16 ans.

⊲ On aurait pu traiter cet exercice avec des if ... else, mais c’est plus élégant avec une autre commande,

switch ... case (merci de vous documenter à son sujet) :

void main() {

int age = readInteger("Entrez l’age de l’enfant");

print("L’enfant a " + age+" ans, il est donc ");

//print ne renvoit pas à la ligne

switch(age) {

case 9 :

case 10 : println("poussin."); //println renvoit à la ligne

break;

case 11 :

case 12 : println("benjamin.");

break;

case 13 :

case 14 : println("minime.");

break;

default : println("cadet.");

break;

}

}

Et le résultat pourra être, dans la console de javascool (avec 2 exécutions, évidemment) :

Compilation réussie !

L’enfant a 9 ans, il est donc poussin.

L’enfant a 12 ans, il est donc benjamin.

8. Modifier cet algorithme pour vérifier que l’âge saisi est un entier n tel que 9 6 n 6 16, et redemande une

saisie sinon.

⊲ Il s’agit ici de blinder la saisie, c’est-à-dire forcer l’utilisateur à saisir un nombre attendu. On peut le faire

avec une boucle tant que (la saisie ne convient pas), comme ici :

void main() {

int age = 20;

while ((age <9) || (age >16)) age = readInteger("Entrez l’age de l’enfant (9-16 ans)");

print("L’enfant a " + age+" ans, il est donc ");

switch(age) {

case 9 :

case 10 : println("poussin.");

break;

case 11 :

case 12 : println("benjamin.");

break;

case 13 :

case 14 : println("minime.");

break;

default : println("cadet.");

break;

}

}

On restera bien dans la boucle while ((age <9) || (age >16)) tant que les nombres ne conviennent pas.

On entre bien dans la boucle puisque int age = 20; ne convient pas.

Pas de problème en cas de saisie de chaînes de caractères ou de nombres non entier : la fonction readInteger()

pré-implémentée dans javascool n’accepte que des entiers. On a bien une saisie blindée.

9. Écrivez un algorithme qui demande un nombre et qui affiche les dix nombres suivants. Par exemple, si

l’utilisateur entre le nombre 17, le programme affichera les nombres 18 à 27.

⊲ Quand on connaît le nombre d’itérations, on utilisera une boucle for dont la syntaxe est

for(initialisation; condition pour laquelle on continue; incrémentation).

Ce qui donne :

void main() {

int i,n = readInteger("Entrez un nombre entier");

for (i = 1; i <= 10; i++) {

println(n+i);

}

}

Un élève a proposé println(n++); dans la boucle, ce qui fonctionne presque, mais donnera 10 entiers

consécutifs en partant de n, et pas de (n+1). Mieux : println(++n); dans la boucle fonctionne parfaitement,

puisque l’incrémentation se fait avant l’affichage !

11. Écrivez un algorithme qui demande un nombre puis calcule sa factorielle.

⊲ C’est un code très proche, mais il faut multiplier et partir de 1.

⊲ Ici, on a blindé la saisie pour avoir n entier naturel. Notez que 0! = 1 par convention.

⊲ Initialiser n = -2; permet de rentrer au moins une fois dans la boucle.

void main() {

int i,factorielle =1,n = -2;

while (n <0) n = readInteger("Entrez un nombre entier positif ");

print(n + "! = ");

for (i = 1; i <= n; i ++) {

factorielle = factorielle * i;

}

println(factorielle);

}

⊲ On teste ce programme et on obtient :

Compilation réussie !

1! = 1

2! = 2

3! = 6

4! = 24

5! = 120

10! = 3628800

11! = 39916800

12! = 479001600

13! = 1932053504

14! = 1278945280

15! = 2004310016

16! = 2004189184

17! = -288522240

On constate le problème à partir de 13! : les nombres de la troisième colonne sont erronés !

⊲ C’est lié aux limitations du type int sur 32 bits (4 octets). Nous en reparlerons, mais disons sommairement

que les nombres sont tronqués à 32 bits en écriture binaire, et que le nombre obtenu pour 17! est interprété

comme un complément à 2 (à suivre), donc négatif !

Le type int est donc clairement limité ici.

12. Écrivez un algorithme permettant à l’utilisateur de saisir les notes d’une classe composées de 20 élèves afin

d’afficher le nombre des ces notes qui sont supérieures à la moyenne de la classe.

⊲ On a besoin de stocker 20 notes, alors pour ne pas déclarer 20 variables, on va déclarer un tableau de 20

variables de type double ainsi : double note[] = new double[20];

⊲ Dans le code, les notes seront donc stockées dans les « cases » numérotées de 0 à 19 : note[0], note[1],

etc jusqu’à note[19].

⊲ On utilise une première boucle for pour la saisie et le calcul de la moyenne, puis une deuxième pour

compter dans la variable nombre les notes supérieures à moyenne.

void main() {

int i,nombre =0;

double moyenne =0;

double note[] = new double[20];

//On déclare ainsi un tableau de 20 notes de type double

for (i =0; i <20; i ++) {

note[i] = readDouble("entrez la note numéro " + i);

moyenne = moyenne+note[i];

println(note[i] + "\t" + moyenne); //On affiche la somme cumulée après une tabulation

}

moyenne = moyenne /20;

for (i =0; i <20; i ++) {

if (note[i] >= moyenne) nombre++;

}

println("Il y a " + nombre+" notes au dessus de la moyenne " + moyenne);

}

⊲ On n’a pas blindé la saisie ici.

⊲ une tabulation se code \t entre guillemets, un retour à la ligne \n, toujours entre guillemets.

13. Écrivez un algorithme qui résout les équations du second degré. Les coefficients du polynôme seront entrés

par l’utilisateur.

void main() {

double a = readDouble("Saisissez le coefficient a du terme de second degré");

double b = readDouble("Saisissez le coefficient b du terme de premier degré");

double c = readDouble("Saisissez la constante c");

double delta = pow(b, 2) -4 * a * c;

print("Le discriminant vaut " + delta);

if (delta <0) println(" < 0 donc pas de racines réelles");

else {

if (delta == 0) println(" donc une unique racine réelle : x = " + ( - b /(2*a)));

else {

print(" > 0 donc deux racines réelles : x_1 = " + (( - b - sqrt(delta)) /(2*a)));

println(" et x_2 = " + (( - b + sqrt(delta)) /(2*a)));

}

}

}

⊲ Rien de bien spécial si ce n’est la fonction double sqrt() ici : racine carrée se dit square root en anglais.

14. Écrivez un algorithme qui calcule le PGCD (Plus Grand Commun Diviseur) de deux nombres entrés par

l’utilisateur en utilisant la méthode d’Euclide.

void main() {

int dividende =0,diviseur =0,temp,reste=1,quotient;

while (dividende <=0) dividende = readInteger("Entrer un premier nombre entier strictement positif

while (diviseur <=0) diviseur = readInteger("Entrer un deuxième nombre entier strictement positif

print("Le PGCD de " + dividende + " et " + diviseur+" est ");

if (diviseur > dividende) {

temp = diviseur;

diviseur = dividende;

dividende = temp;

//on veut que le plus grand soit le dividende.

}

while (reste > 0) {

reste = dividende % diviseur;

quotient = (dividende-reste) / diviseur;

dividende = diviseur; //prépare la prochaine division euclidienne

diviseur = reste; //de même et du coup ...

}

println(dividende);

// ... c’est dividende qui contient le dernier diviseur non nul qui est le PGCD

}

On dit aussi que reste vaut a modulo b.
